# DeepGYM 产品说明文档

版本：0.1
状态：动作指导、训练中心、首个正式搭档、账号与云同步 MVP 已实现；等待完整跨设备端到端联调
更新日期：2026-07-22

## 1. 产品一句话

DeepGYM 是一个运行在 macOS 和 Windows 上的私人训练伴侣：帮助健身者看懂动作、记录训练，并与会回应真实训练成果的虚拟搭档长期共同成长。

## 2. 要解决的问题

健身者在训练现场经常遇到三个连续问题：

- 不确定动作怎么做，担心姿势错误；
- 训练中记录散落在备忘录、纸张或不同应用里；
- 训练结束后不知道今天练了什么、总量多少、是否比上次进步。

DeepGYM 把“动作指导 → 训练执行 → 当日总结”放进一个本地优先的桌面产品里，降低查找和记录的成本。

## 3. 目标用户

首要用户是已有基础训练习惯、每周训练 2–5 次的健身者。他们会使用自由重量、固定器械或自重动作，但未必记得每个动作的标准步骤，也希望持续记录重量与次数。

暂不以专业教练、康复治疗师或竞技运动员为首要用户；后续可以为他们增加训练模板、学员管理或更细的周期计划。

## 4. 核心体验

```text
创建计划或查找动作 → 开始训练 → 记录每一组 → 结束训练 → 查看今日总结 → 从日历回顾
```

产品设计原则：

- 训练现场优先：少输入、按钮大、信息层级清晰；
- 正确性优先：动作步骤、目标肌群、器械与安全提示要容易看到；
- 本地优先：登录设备上的核心记录离线可用；首次使用必须注册或登录；
- 不冒充医疗建议：疼痛、伤病和康复问题必须提示咨询专业人士；
- 数据可带走：后续支持导出训练记录，避免数据被产品锁定。

## 5. 功能范围与阶段

### 阶段 1：动作指导（已完成）

目标：用户能在 10 秒内找到动作，并理解如何完成。

包含：

- 动作库列表与分页/渐进加载；
- 按身体部位、器械筛选；
- 按动作名和目标肌群搜索；
- 动作详情：动图、中文步骤、器械、主要/辅助肌群；
- 动图暂停与继续；
- 媒体授权元数据保留在数据文件中，正式界面按已取得的许可不展示外链署名；
- 加载失败和无结果提示。

当前阶段的限制：

- 上游数据提供中文动作说明，但没有完整的中文动作名称；当前版本先为常见动作提供中文名称，其余保留英文名；
- 已选择媒体方案 A，动图通过互联网从固定上游版本加载，不制作离线媒体包；
- 正式桌面应用已迁移到 Tauri 2；macOS 未签名测试包已验证，Windows 安装包仍需在 Windows 环境构建验证。

### 阶段 2：训练记录与今日总结（MVP 已完成）

目标：用户完成训练后，一眼看懂当天成果。

第一版已包含：

- 将动作加入“今日训练”；
- 每组记录重量、次数和完成状态；
- 默认准备三组，并支持增删组、移除动作；
- 输入后自动保存进行中的训练；
- 结束训练并生成总结；
- 总结指标：训练时长、动作数、总组数、总次数、训练容量（重量 × 次数）；
- SQLite 本地历史记录与按训练日期查看。

后续版本再加入备注、复制上一组/上次训练、休息计时、涉及部位和个人最好成绩。这些能力不阻塞第一版闭环。

### 阶段 3：周期计划与训练数据（MVP 已完成）

- 侧栏将原“今日训练”和“训练记录”合并为“训练”，内部包含训练计划、训练记录、训练数据；
- 用户可保存多套周期计划，同时只启用一套；
- 计划按星期设置训练日，并为每个动作设置目标组数和次数范围；
- 从计划开始训练时自动创建动作和空白组，已有训练只合并缺失动作；
- 计划来源和次数目标快照进入本次训练，后续修改计划不会改写历史；
- 日历以中文、周一为首日，支持月份与年份下拉；有完成训练的日期显示绿色圆点；
- 点击日期在应用内窗口中查看同日全部已完成训练、汇总指标和逐组表格；
- 训练计划和软删除通过本地 outbox 自动同步 Supabase。

后续版本再加入动作趋势、训练容量趋势、周/月回顾和更完整的数据导入导出。

### 阶段 4：健身搭档（核心闭环 MVP 已实现）

- 侧栏新增一级“搭档”，包含选择、详情、创建、主页、成长、时间轴、切换管理和设置；
- 正式角色目录已加入首个角色“力力兔”，使用用户提供的六阶段形象；
- 搭档采用六阶段、至少 84 天的成长周期，每次进化同时要求时间、成长值和训练次数；
- 成长值来自完成训练、有效组数、有效时长、计划完成、新动作与个人进步，并设每日 120、每周 420 上限；
- 高 RPE 不直接增加成长值，暂停训练不扣分、不降级；删除训练会修正成长账本，但已解锁形态不倒退；
- 训练记录增加“开始本组”、休息计时、暂停与提前结束，搭档动作按 12 类动作族同步；
- 训练结束收集可跳过的 RPE 与感受，次日可记录恢复状态；疼痛反馈只提供停止训练和咨询专业人士的安全提醒；
- 训练结算展示成长来源、羁绊变化和进化仪式；图鉴、装扮与纪念卡分享暂为锁定入口；
- “力力兔”每个阶段都包含透明静态形象、待机/休息/庆祝/恢复动画和 12 类训练动作动画；系统减少动态效果开启时自动回退为静态图；
- 搭档数据本地优先保存到 SQLite，并通过 Supabase RLS 同步到当前账号。

后续角色素材接入契约继续包含每阶段静态图/待机动画、12 类动作动画、互动动画、进化素材和最终形态剪影。首版不包含语音、穿戴设备、摄像头识别、付费成长值或排行榜。

### 账号与云同步（MVP 已实现）

- 邮箱 + 密码注册登录，应用内输入 6 位注册或恢复验证码；
- 会话只通过系统凭据管理器保存，不使用普通 localStorage；
- 训练始终先写本机 SQLite，联网后自动同步 Supabase；
- 首次登录先恢复云端数据，再将旧版无账号记录归入当前账号；
- 不同账号的数据按 `userId` 隔离，云端由 RLS 强制只允许本人访问；
- 多设备产生两份进行中训练时由用户选择，其余保留为未完成历史；
- “我的”支持修改用户昵称和上传头像；头像存入用户自己的私有云端目录；
- 退出登录前必须完成同步；删除账号会删除 Auth、云端数据、本机副本和凭据。

## 6. 阶段 1 用户故事

- 作为训练者，我可以输入“卧推”或 `bench press` 找到对应动作；
- 作为训练者，我可以只看“胸部 + 杠铃”动作；
- 作为训练者，我可以打开动作详情并看到循环动图；
- 作为训练者，我可以暂停动图，逐步阅读中文说明；
- 作为训练者，我能明确看到目标肌肉、辅助肌肉和所需器械；
- 作为产品所有者，我能确认媒体加载方式和数据中的授权元数据符合已取得的许可。

## 7. 阶段 1 验收标准

1. 本地数据包含上游 1,324 条动作记录，启动时能完整解析；
2. 搜索和筛选可组合使用，清除条件后恢复完整结果；
3. 首屏不同时请求全部媒体，只为当前可见卡片加载缩略图；
4. 点击卡片能打开详情，显示中文步骤；若中文缺失则明确回退英文；
5. 详情动图可暂停/继续，图片失败时仍能阅读文字指导；
6. 键盘可以聚焦搜索、打开卡片、关闭详情；
7. 窄窗口下仍能完成搜索、筛选和查看详情；
8. 正式 React 应用不显示逐动图或页脚媒体外链；
9. 自动检查验证记录数量、必填字段和内部来源元数据没有丢失。

## 8. 训练记录 MVP 验收标准

1. 用户可从任意动作详情将动作加入今日训练，重复加入不会产生重复动作；
2. 每个动作默认有三组空白记录，可填写非负重量和正整数次数，并可增删组；
3. 只有填写有效次数并标记完成的组，才会计入总结；
4. 进行中训练的输入会自动写入本机存储；
5. 至少完成一组后才能结束训练；
6. 结束训练后立即显示当天动作数、组数、总次数、训练容量和累计时长；
7. 训练数据日历可按本地日期查看同日每次训练和每个已完成组；
8. Tauri 桌面应用使用 SQLite；浏览器开发预览仅使用 localStorage，不代表正式桌面存储；
9. 数据库通过版本化迁移自动创建，应用升级不依赖用户手工建表。

## 9. 周期计划与训练数据验收标准

1. 主导航只显示一个“训练”入口，默认进入“训练记录”；
2. 用户可创建、编辑、复制、启用、停用和删除计划，同时只有一套当前计划；
3. 计划可设置每周训练日、动作顺序、目标组数与次数范围；
4. 从计划开始训练会创建相应空白组，合并进行中训练时不覆盖已有输入；
5. 日历以中文显示、周一为每周第一天，跨时区查询不会偏移日期；
6. 只有已完成且未删除的训练出现在日历和日数据表格；
7. 同一天多场训练全部展示，记录删除前需要应用内二次确认；
8. 计划支持离线增删改，联网后按父子顺序同步并受 RLS 账号隔离。

## 10. 非目标

第一版不包含：

- 摄像头姿态识别或自动纠错；
- AI 医疗、伤病诊断或康复处方；
- 饮食记录、卡路里管理；
- 教练/学员后台；
- 社交动态和排行榜；
- 付费订阅；
- 社交登录、手机号登录；
- 管理员后台。

## 11. 数据与隐私

训练记录先保存在用户电脑的 SQLite 数据库，登录且联网后同步到用户自己的 Supabase 账号。桌面会话保存在 macOS Keychain 或 Windows Credential Manager；客户端只包含 publishable key，删除账号所需的服务端 secret 只存在于 Edge Function 环境中，SMTP 凭据只保存在 Supabase 加密配置中。认证邮件通过已验证的 `auth.deepgymapp.com` 子域名发送。公开发布前仍需补充隐私政策、数据导出说明和数据处理条款审查。当前不采集遥测数据。

## 13. 成功指标（产品上线后）

- 搜索至打开动作详情的中位时间低于 10 秒；
- 搜索无结果率低于 5%；
- 动作详情打开后加入训练的转化率；
- 开始训练后完成并生成总结的比例；
- 每周至少记录 2 次训练的活跃用户比例；
- 崩溃率、数据丢失反馈数和媒体加载失败率。
