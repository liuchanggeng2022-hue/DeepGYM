# DeepGYM 产品说明文档

版本：0.1
状态：产品方向已定义，桌面技术栈已选择 Tauri 2 + React + TypeScript
更新日期：2026-07-21

## 1. 产品一句话

DeepGYM 是一个运行在 macOS 和 Windows 上的私人训练伴侣：帮助健身者看懂动作、完成训练、记录每一组，并在当天训练结束后得到清晰的训练总结。

## 2. 要解决的问题

健身者在训练现场经常遇到三个连续问题：

- 不确定动作怎么做，担心姿势错误；
- 训练中记录散落在备忘录、纸张或不同应用里；
- 训练结束后不知道今天练了什么、总量多少、是否比上次进步。

DeepGYM 把“动作指导 → 训练执行 → 当日总结”放进一个本地优先的桌面产品里，降低查找和记录的成本。

## 3. 目标用户

首要用户是已有基础训练习惯、每周训练 2–5 次的健身者。他们会使用自由重量、固定器械或自重动作，但未必记得每个动作的标准步骤，也希望持续记录重量与次数。

暂不以专业教练、康复治疗师或竞技运动员为首要用户；后续可以为他们增加训练模板、学员管理或更细的周期计划。

## 4. 核心体验

```text
查找动作 → 查看动图与步骤 → 加入今天的训练 → 记录每一组 → 结束训练 → 查看今日总结
```

产品设计原则：

- 训练现场优先：少输入、按钮大、信息层级清晰；
- 正确性优先：动作步骤、目标肌群、器械与安全提示要容易看到；
- 本地优先：核心记录离线可用，用户不登录也能开始；
- 不冒充医疗建议：疼痛、伤病和康复问题必须提示咨询专业人士；
- 数据可带走：后续支持导出训练记录，避免数据被产品锁定。

## 5. 功能范围与阶段

### 阶段 1：动作指导（当前）

目标：用户能在 10 秒内找到动作，并理解如何完成。

包含：

- 动作库列表与分页/渐进加载；
- 按身体部位、器械筛选；
- 按动作名和目标肌群搜索；
- 动作详情：动图、中文步骤、器械、主要/辅助肌群；
- 动图暂停与继续；
- 媒体版权署名；
- 加载失败和无结果提示。

当前阶段的限制：

- 上游数据提供中文动作说明，但没有完整的中文动作名称；当前版本先为常见动作提供中文名称，其余保留英文名；
- 已选择方案 A，动图通过互联网从固定上游版本加载，不制作离线媒体包；
- 正式桌面应用已迁移到 Tauri 2；macOS 未签名测试包已验证，Windows 安装包仍需在 Windows 环境构建验证。

### 阶段 2：训练记录与今日总结

目标：用户完成训练后，一眼看懂当天成果。

计划包含：

- 将动作加入“今日训练”；
- 每组记录重量、次数、完成状态和备注；
- 快速复制上一组、上一训练日数据；
- 训练计时与组间休息计时；
- 结束训练并生成总结；
- 总结指标：训练时长、动作数、总组数、总次数、训练容量（重量 × 次数）、涉及部位、个人最好成绩；
- 本地历史记录与按日期查看。

### 阶段 3：进步与计划

- 动作趋势和训练容量趋势；
- 自定义训练模板；
- 周/月训练回顾；
- 数据导入导出；
- 可选云同步和多设备使用。

## 6. 阶段 1 用户故事

- 作为训练者，我可以输入“卧推”或 `bench press` 找到对应动作；
- 作为训练者，我可以只看“胸部 + 杠铃”动作；
- 作为训练者，我可以打开动作详情并看到循环动图；
- 作为训练者，我可以暂停动图，逐步阅读中文说明；
- 作为训练者，我能明确看到目标肌肉、辅助肌肉和所需器械；
- 作为产品所有者，我能确认每处媒体使用都带有来源署名。

## 7. 阶段 1 验收标准

1. 本地数据包含上游 1,324 条动作记录，启动时能完整解析；
2. 搜索和筛选可组合使用，清除条件后恢复完整结果；
3. 首屏不同时请求全部媒体，只为当前可见卡片加载缩略图；
4. 点击卡片能打开详情，显示中文步骤；若中文缺失则明确回退英文；
5. 详情动图可暂停/继续，图片失败时仍能阅读文字指导；
6. 键盘可以聚焦搜索、打开卡片、关闭详情；
7. 窄窗口下仍能完成搜索、筛选和查看详情；
8. 页面明确显示 `© Gym visual — https://gymvisual.com/`；
9. 自动检查可以验证记录数量、必填字段和署名未被移除。

## 8. 非目标

第一版不包含：

- 摄像头姿态识别或自动纠错；
- AI 医疗、伤病诊断或康复处方；
- 饮食记录、卡路里管理；
- 教练/学员后台；
- 社交动态和排行榜；
- 账号、付费和云同步。

## 9. 数据与隐私

训练记录计划默认只保存在用户电脑上。若未来加入云同步，需要单独确认账号体系、隐私政策、数据删除与导出机制。在没有用户明确选择前，不采集健康数据或遥测数据。

## 11. 成功指标（产品上线后）

- 搜索至打开动作详情的中位时间低于 10 秒；
- 搜索无结果率低于 5%；
- 动作详情打开后加入训练的转化率；
- 开始训练后完成并生成总结的比例；
- 每周至少记录 2 次训练的活跃用户比例；
- 崩溃率、数据丢失反馈数和媒体加载失败率。
